# デザインレビュー

このコマンドは、UIコンポーネントやフロントエンドコードのデザイン品質をレビューします。

## サブエージェント呼び出し

**重要**: このコマンドは `design-reviewer` サブエージェントを使用します。

design-reviewer サブエージェントを使用して、以下のデザイン品質をレビューしてください。

## レビュー観点

### 1. アクセシビリティ（a11y）
- WCAG 2.1 ガイドラインへの準拠
- キーボード操作のサポート
- スクリーンリーダー対応
- カラーコントラスト比の確認
- フォーカスインジケーターの視認性

### 2. レスポンシブデザイン
- モバイル、タブレット、デスクトップでの表示確認
- ブレークポイントの適切性
- フレキシブルレイアウトの実装
- タッチターゲットのサイズ（最小44x44px）

### 3. デザインシステムとの整合性
- カラーパレットの使用
- タイポグラフィの統一
- スペーシングの一貫性
- コンポーネントの再利用

### 4. UXベストプラクティス
- ユーザーフィードバックの適切性
- エラーメッセージの明確性
- ローディング状態の表示
- 操作の直感性

## 参照スキル

スキル `design-review` のガイドラインに従ってレビューを実施してください。

## 使用例

```
/michi:design-review
```

コマンド実行後、現在のプロジェクトのUIコンポーネントをレビューし、改善提案を行います。

## 出力

レビュー結果は以下の形式で出力されます：

1. **レビューサマリー**: 全体的な評価
2. **問題点**: 発見された問題のリスト（優先度付き）
3. **改善提案**: 具体的な改善策
4. **参考資料**: 関連するベストプラクティスへのリンク

## 言語設定

出力言語は .kiro/project.json の設定に従います。
